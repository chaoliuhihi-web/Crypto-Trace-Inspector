数字货币痕迹检测系统
UI 设计

一、设计目标

本方案核心目标：

强化流程感（执法步骤清晰）

强化证据链可信表达

提升工具专业性与权威感

降低产品化气息

提高信息密度与可追溯性

符合内部执法系统视觉语境

二、整体架构结构

采用 工作站式三段结构

┌────────────────────────────────────────────────────────────┐
│ 顶部信息栏（案件信息 · 操作员 · 证据链状态 · 系统状态）     │

│ 底部运行状态栏（任务 · 进度 · 日志 · 版本 · Schema）        │
└────────────────────────────────────────────────────────────┘

八、系统特征表达重点

本重构版本强调三点视觉语言：

证据不可篡改感

审计链完整感

操作留痕感

通过以下方式体现：

每页显示操作ID

全局证据链状态

日志常驻展示

规则版本明确

报告封面标准化
